# 样式文件说明

## 文件结构

- `common.css` - 公共样式文件，包含所有页面共用的基础样式
- `history.css` - 历史记录页面特定样式
- `options.css` - 配置页面特定样式  
- `popup.css` - 弹出窗口特定样式
- `edit-dialog.css` - 改图对话框特定样式

## 公共样式 (common.css)

### 包含内容
- CSS变量定义（颜色、尺寸、阴影等）
- 基础重置样式
- 通用字体和布局
- 按钮样式系统
- 表单元素样式（输入框、下拉框、复选框等）
- 卡片样式
- 通知样式
- 模态框基础样式
- 图片相关样式
- 动画定义
- 滚动条样式
- 响应式基础样式
- Chrome插件特定样式

### CSS变量系统
使用CSS自定义属性定义了完整的设计系统：
- 颜色系统：主色、背景色、文字色、状态色
- 尺寸系统：圆角、阴影、间距
- 动画系统：过渡效果

## 页面特定样式

### history.css
- 历史记录页面的网格布局
- 图片卡片样式
- 搜索和筛选工具栏
- 图片预览模态框

### options.css  
- 配置页面的侧边栏导航
- 表单布局和样式
- 服务商卡片管理
- 模板管理界面

### popup.css
- 弹出窗口的固定尺寸布局
- 图片上传和预览
- 结果展示区域

### edit-dialog.css
- 改图对话框的居中布局
- 渐变背景
- 表单样式覆盖

## Chrome插件兼容性

### 特殊考虑
- 使用相对路径引用CSS文件
- 避免使用@import，改用HTML中的多个link标签
- 确保弹出窗口的固定尺寸
- 处理内容脚本的样式隔离

### 样式隔离
- 使用`.chrome-extension-content`类来重置样式
- 避免全局样式污染页面内容

## 使用方法

### HTML文件中的引用顺序
```html
<link rel="stylesheet" href="styles/common.css">
<link rel="stylesheet" href="styles/[page-specific].css">
```

### 样式覆盖
页面特定样式可以覆盖公共样式，通过：
1. 更高的特异性
2. 相同特异性但后加载
3. 使用CSS变量进行主题定制

## 维护指南

### 添加新样式
1. 优先考虑是否应该添加到common.css
2. 使用已定义的CSS变量
3. 遵循现有的命名约定
4. 考虑响应式设计

### 修改现有样式
1. 检查是否影响其他页面
2. 优先修改CSS变量而不是具体值
3. 测试所有相关页面

### 性能优化
- 避免重复的样式定义
- 使用CSS变量减少重复值
- 合理使用选择器特异性
- 避免过深的嵌套
